1. Datos, información y conocimiento

De dato a información… y a conocimiento

En el lenguaje coloquial usamos «dato» e «información» como sinónimos, pero en digitalización NO son lo mismo.

Dato

  • Hechos, eventos o valores sin procesar.
  • Por sí solos no tienen significado ni ayudan a decidir.
  • Ejemplo: «5 %».

Información

  • Conjunto de datos procesados y organizados.
  • Tienen sentido para el receptor y sirven para decidir.
  • Ejemplo: «Las ventas han subido un 5 % este mes».

¿Cómo se convierte el dato en información?

  • Contextualizar: añadir tiempo, lugar, causa, etc.
  • Categorizar: agrupar datos similares.
  • Calcular: aplicar operaciones y estadísticas.
  • Condensar: resumir en tablas, gráficos o resúmenes.

Información → Conocimiento

La información se transforma en conocimiento cuando…

  • Se compara con otros momentos o grupos.
  • Se buscan causas y consecuencias.
  • Se relaciona con otros datos e ideas.
  • Se comparte y se contrasta con otras personas.
Clave de examen:
Datos = materia prima · Información = datos procesados · Conocimiento = información interpretada para tomar decisiones.
2. Ciclo de vida de los datos

Desde que nace un dato hasta que se borra

Los datos siguen un ciclo de vida con varias fases. Es importante gestionarlas bien para evitar pérdida, desorden o problemas de seguridad.

Fases del ciclo de vida

  1. Captura: generación o recogida de los datos (formularios, sensores, bases de datos…).
  2. Almacenamiento: guardado en bases de datos, ficheros locales o nube.
  3. Procesamiento y visualización: limpieza, análisis, dashboards, gráficos…
  4. Uso operativo: la empresa usa la información para decidir y mejorar procesos.
  5. Almacenamiento / eliminación / reutilización: se archivan, destruyen o reutilizan según políticas y normativa.

Ejemplo rápido

Un taller mecánico registra los datos del vehículo (captura), los guarda en su sistema (almacenamiento), analiza históricos para ver averías frecuentes (procesamiento), organiza citas y piezas (uso operativo) y finalmente archiva o borra datos antiguos (archivo/eliminación).

3. Big Data

Qué es y cuáles son sus 5 V

El Big Data se refiere a conjuntos de datos tan grandes, rápidos y variados que no se pueden tratar con herramientas tradicionales.

Las 5 V del Big Data

  • Volumen: enorme cantidad de datos.
  • Velocidad: se generan y procesan muy rápido (tiempo real).
  • Variedad: muchos formatos (texto, vídeo, imagen, sensores…).
  • Veracidad: importancia de que los datos sean fiables.
  • Valor: las conclusiones deben ser útiles para la empresa.

Desventajas del Big Data

  • Dificultad para gestionar tanta cantidad de datos.
  • Falta de personal cualificado en análisis de datos.
  • Riesgos de ciberseguridad y robos de información.
  • Costes elevados (infraestructura y especialistas).
Recuerda: a veces se habla de hasta 10 V (visualización, vulnerabilidad, volatilidad, validación, variabilidad…), pero las que siempre debes saber son: volumen, velocidad, variedad, veracidad y valor.
4. Aplicaciones del Big Data

Big Data en distintos sectores profesionales

Casi todos los sectores utilizan análisis masivo de datos para detectar patrones, predecir situaciones y tomar mejores decisiones.

Ejemplos por sectores

  • Sanidad: detección precoz de enfermedades, selección de tratamientos.
  • Deporte: wearables que relacionan actividad física con salud.
  • Seguridad informática: detección de patrones anómalos y ciberataques.
  • Ciudades inteligentes: tráfico, transporte público, consumo de recursos.
  • Educación: detectar riesgo de abandono escolar, mejorar evaluación.
  • Sector financiero: análisis de riesgos, detección de fraudes.
  • Turismo: conocer destinos y servicios preferidos por los clientes.
  • Mercado inmobiliario: prever zonas de interés y evolución de precios.
  • Investigación: decodificación del ADN, avances científicos.

Caso Uber (resumen)

  • Recoge datos de viajes, conductores y usuarios.
  • Predice la demanda y coloca conductores donde hacen falta.
  • Optimiza rutas y tiempos de espera.
  • Aplica precios dinámicos según oferta y demanda.
Clave: en exámenes pueden pedirte que relaciones un sector (sanidad, banca, turismo…) con un uso concreto del Big Data.
5. Big Data, ML, IA y Deep Learning

Cómo encajan todas las piezas

El Big Data está muy relacionado con el análisis de datos, el machine learning, el deep learning y la inteligencia artificial.

Big Data y análisis de datos

  • Big Data: aporta la infraestructura para almacenar y procesar datos masivos.
  • Análisis de datos: se centra en inspeccionar, limpiar y transformar los datos para extraer conclusiones.

Machine Learning y Deep Learning

  • Machine learning: algoritmos que aprenden patrones sin ser programados explícitamente.
  • Deep learning: tipo avanzado de ML que usa redes neuronales para tareas complejas (imágenes, lenguaje…).
  • Necesitan muchos datos → el Big Data es su «combustible».

Inteligencia artificial (IA)

  • Sistemas capaces de realizar tareas que requieren inteligencia humana.
  • Usan modelos de ML y deep learning entrenados con Big Data.
Idea resumen:
Big Data (datos) → Análisis de datos (conclusiones) → ML / Deep Learning (modelos que aprenden) → IA (sistemas que toman decisiones).
6. Ciencia de datos

Del problema al modelo y a la acción

La ciencia de datos es un campo multidisciplinar que usa estadística, programación, visualización y ML para extraer conocimiento útil de los datos.

Fases principales

  1. Definición del problema: qué queremos mejorar o predecir.
  2. Adquisición y preparación de datos: recopilar, limpiar y dejar listos los datos.
  3. Análisis exploratorio (EDA): visualizar, buscar patrones y relaciones.
  4. Modelado: crear modelos predictivos o clasificadores.
  5. Implementación y evaluación: aplicar el modelo en la práctica y medir resultados.

Ejemplo del libro: un bufete de abogados analiza duración y tipo de casos, construye un modelo para predecir la duración y ajusta la asignación de recursos según los resultados.

Para repasar rápido: recuerda esta secuencia:
Problema → Datos → Exploración → Modelo → Implantación y evaluación.
